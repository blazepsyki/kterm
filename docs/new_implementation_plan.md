# 🚀 구현 계획: KTerm 어드밴스드 UI/UX 라우팅 및 탭 관리 고도화 (Phase 10)

기나긴 터미널 코어 파이프라인과 ConPTY 버그 해결 여정이 무사히 끝났습니다. 이제 과거 `v2`, `v3` 기획 단계에서 제안되었으나 보류되었던 **프리미엄 네이티브 터미널 지향의 고급 UI 기능**들(화면 분할, 탭/헤더 통합, 테마 시스템)을 본격적으로 개발하기 위한 새로운 통합 기획서입니다.

---

## 🏗️ 1. 레이아웃 및 탭 분할 (Split View System)

### 1-1. 화면 분할 (Drag to Split)
- **개념**: 단일 탭 화면에서 특정 탭이나 액션 핸들을 마우스로 드래그하여 화면 가장자리(상/하/좌/우)에 놓으면, 해당 방향으로 터미널이 분할(Split) 다중 렌더링되게 합니다.
- **분할 경계선 리사이징**: 나누어진 각 Pane들 사이에 드래그 가능한 분할 선(Sash) 영역을 두어 자유롭게 비율을 조정합니다.
- **포커스 라우팅**: 여러 개의 SSH 세션 화면이 띄워진 상태에서 마우스 클릭이나 단축키(e.g., `Alt+방향키`)를 통해 현재 입력(Active Session) 포커스를 결정합니다.

---

## 🖱️ 2. 터미널 관리 및 세션 네비게이션 고도화

### 2-1. 탭 순서 변경 (Tab Reordering)
- 상단 탭 창에서 탭 레이블을 드래그 앤 드롭(Drag & Drop)하여 탭 간의 순서를 동적으로 변경합니다.

### 2-2. 사이드바 세션 트리 (Session Tree Management)
- 여러 서버를 등록해놓고 클릭 한 번으로 새 탭 세션을 띄울 수 있는 좌측 사이드바 트리(폴더-자식 구조) UI 컴포넌트를 이식합니다.
- (향후 개발될 Telnet, Serial 세션 프로필 등을 여기서 관리합니다.)

---

## 🎨 3. 통합 커스텀 헤더 및 디자인 업그레이드

### 3-1. 탭 바 - 타이틀 바 병합 로직
- 현재 상단에 커스텀 타이틀(닫기 버튼) 1줄, 그 아래에 탭 바 1줄을 쓰고 있습니다. 
- 이를 크롬 브라우저처럼 **최상단 타이틀 드래그 바에 탭 바를 완전히 병합**시켜 시각적 수직 공간 효율을 극대화합니다.

### 3-2. 헤더 세션 브레드크럼 및 검색창
- 헤더 중앙 영역에 현재 연결된 세션의 상태(`username@host`) 및 핑(Ping)/암호화 정보를 브레드크럼 뱃지 형식으로 알기 쉽게 표시합니다.
- 헤더 우측 또는 단축키로 켤 수 있는 **커맨드 팔레트/통합 검색 기능** 상자를 추가 검토합니다.

### 3-3. 미적(Aesthetic) 글래스모피즘(Glassmorphism) 테마 엔진
- 단순 무채색 배경 대신, 운영체제의 아크릴/블러 반투명(Glass/Blur) 효과를 이식하거나 고급스러운 그라데이션 다크/라이트 체인지를 지원하는 **테마 엔진 렌더러**를 추가합니다.

---

## ⚙️ 4. 사용자 환경 설정창 (Settings Component)

현재 기준으로 Settings의 베이스라인은 이미 구현되어 있습니다.

- 구현 완료
	- Preferences/Theme 설정 탭 UI
	- 프로토콜별 주요 옵션(SSH/Telnet/Serial/Local/RDP/VNC) 조절
	- JSON 영속화: 실행 파일 옆 `kterm_settings.json`
	- 기본값과 다른 설정만 저장, 앱 시작 시 자동 로드

- 후속 고도화 과제
	- 숫자/열거 입력 필드 검증 강화
	- 프로파일 단위(서버별) 설정 저장
	- 테마/폰트/커서/벨 등 시각·오디오 설정 확장

---

> [!NOTE]
> 본 기획안은 이전에 완수되지 못한 V2/V3의 남은 작업들을 계승하며 가장 최우선적으로 돌입할 UI 개발 청사진입니다. 사용자의 승인이 떨어지면, 윈도우 레이아웃 아키텍처를 뒤엎는 **1번 탭 분할(Split View)** 또는 **3번 헤더 통합** 작업부터 순차 진행할 계획입니다.
