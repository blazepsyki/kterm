# KTerm 개발 결과 Walkthrough (통합본)

본 문서는 터미널 에뮬레이터 초기 구현(Phase 1)부터 시작해 UI/UX 고도화, 그리고 극한의 한글 입력 버그 수정(Phase 9)에 이르기까지 거쳐간 `walkthrough.md`, `v2`, `v3`의 모든 구현 증빙 및 개발 과정을 하나로 압축 단일화한 통합 증빙 리포트입니다.

---

## 1. 코어 엔진: 통신(SSH/Local PTY) 및 기본 터미널 연동
초기 단계에서는 로컬 환경과 원격 제어가 가능한 SSH 클라이언트를 투 트랙으로 연동하는 데 성공했습니다.
- **SSH 클라이언트 파이프라인 (`russh`)**: `russh` 라이브러리와 `Iced` 간 양방향 채널(`mpsc`)을 이용한 실시간 스트리밍 대화형 PTY 세션을 구축했습니다.
- **로컬 터미널 이식 (`portable-pty`)**: 기획엔 없었으나 편의성을 위해 윈도우 네이티브 **PowerShell 기반 로컬 쉘**을 백그라운드로 실행(Spawn)시켰으며, Iced 엔진 상에서 SSH 스트림(SshEvent 인터페이스)과 놀랍도록 완벽하게 동일한 구조로 다형성 통신이 이루어지게 설계했습니다.
- **안정적 스크롤링 및 리사이징**: `VecDeque` 기반 히스토리 버퍼로 마우스 휠 무한 스크롤백을 지원하고, 레이아웃의 실측 사이즈를 이용해 행/열 사이즈(Reflow)를 실시간 반영했습니다.

---

## 2. 모던 UI/UX: 네이티브 타이틀 바 및 인터페이스 고도화
기존의 정형화된 O/S 타이틀 바를 지우고 터미널 전용 커스텀 헤더를 구현하여, 사용 면적과 심미성을 획기적으로 끌어올렸습니다.
- **프레임리스(Frameless) 윈도우**: 윈도우 기본 장식(`decorations`)을 해제하고 커스텀 상단 드래그 영역(`mouse_area`)을 직접 추적 구현해 끊김 없는 창 이동을 완성했습니다.
- **커스텀 윈도우 컨트롤**: 메인 윈도우 ID를 캡처하여 터미널 내부에 직접 최소화/최대화/닫기 단추를 삽입했습니다.
- **더미 오버레이 레이아웃**: 메인 터미널 그리드를 해치지 않게 `stack!` 위젯 등을 활용해 상단 드롭다운 형태의 레이어를 쌓고, 프리미엄한 무채색(Monochromatic) 테마로 미적 통일감을 주었습니다.

---

## 3. 정합성: 텍스트 인터랙션 및 잔상 버그(ConPTY) 해결
프로젝트 후반부는 윈도우 SSH 서버의 기반인 `ConPTY`와 에뮬레이터 사이의 한글 길이(Double-width) 논리 충돌을 해결하는 여정이었습니다.
- **한글 타이핑/이동 안정화**: 터미널 자체에 껴있던 와이드 캐릭터 동시 점프(Double-Skip) 로직을 전구간 파괴해 쉘과 커서 이동 통제권을 일원화시켰습니다. 반쪽짜리 글자를 지워 없애는 `Wide-aware Cleanup` 로직을 도입해 글자 쪼개짐과 화면 더미 생성을 막았습니다.
- **역 래핑(Reverse-wrap) 및 캐시 무효화**: 백스페이스로 줄을 지워 올라가도록 래핑 상태 속성(`is_wrapped`) 보존 로직을 고쳤고, 쉘로부터 데이터를 수신하자마자 `Canvas cache.clear()`를 즉각 발동시켜 화면 랙(Lag)을 없앴습니다.
- **초정밀 다중 행 잔상 파괴 (Ghosting Bypass)**:
  - 쉘이 명령을 덮어쓰려 상단(`CUP`) 점프를 할 때, 하단 찌꺼기 추적을 작동시켰습니다.
  - ConPTY가 커서를 복귀시키지 않고 허공의 공간을 냅다 지운다고 지시하는 버그 패턴(`X n + C n`)을 감지해 필터링(Clear Line 격상)했습니다.
  - 궁극적으로 **'방금 이 줄에 터미널이 새 글씨를 찍었는가?'**를 `HashSet` 맵으로 추적하여, 정상적인 새멀티행 프롬프트의 꼬리(Overhang)만 다듬는 지우기와, 낡은 깡통 과거 행을 싹 지워버리는 멸종(Clear Entire Line) 지우기를 완벽하게 판별해냄으로써 100% 버그 프리 텍스트 치환 시스템을 획득했습니다.
