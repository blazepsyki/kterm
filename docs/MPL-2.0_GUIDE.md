# MPL-2.0 안내 (kterm)

이 문서는 kterm에서 확인된 MPL-2.0 라이선스 의존성에 대해, 배포 시 실무적으로 확인해야 할 포인트를 정리합니다.

- 작성 목적: 내부 컴플라이언스 체크 참고
- 법적 고지: 이 문서는 법률 자문이 아니며, 최종 판단은 조직의 법무/컴플라이언스 검토를 따릅니다.

## 1. MPL-2.0 핵심 요약

MPL-2.0은 파일 단위 copyleft 성격을 갖는 라이선스입니다.

- MPL 적용 파일 자체를 수정해 배포하는 경우:
  - 해당 수정 파일의 소스 공개 의무가 생깁니다.
- MPL 비적용 파일(우리 애플리케이션 고유 코드)까지 자동으로 MPL로 전염되지는 않습니다.
- 라이선스 고지와 원문 제공(또는 접근 경로 제공)은 유지해야 합니다.

실무적으로는 "MPL 파일을 수정했는지"가 가장 중요한 분기점입니다.

## 2. kterm 현재 맥락

현재 문서 기준으로 직접 의존성 중 MPL-2.0 항목은 아래와 같습니다.

- serialport 4.9.0: MPL-2.0

참고 문서:

- 루트 라이선스 목록: ../THIRD_PARTY_LICENSES.md

## 3. 배포 시 체크리스트

### A. 기본 고지

- [ ] 제3자 라이선스 문서에 MPL-2.0 의존성을 명시했다.
- [ ] MPL-2.0 라이선스 원문(또는 공식 URL) 접근 경로를 제공했다.
- [ ] 배포 아카이브/설치본/문서에서 제3자 라이선스 위치를 안내했다.

### B. 수정 여부 확인

- [ ] MPL-2.0 대상 크레이트 소스를 직접 수정하지 않았다.
- [ ] (수정했다면) 수정한 MPL 파일의 소스와 변경 내역 제공 방안을 준비했다.

### C. 릴리즈 검증

- [ ] `cargo deny check licenses` 결과를 릴리즈 증적에 첨부했다.
- [ ] `cargo metadata --locked` 기준 라이선스 목록을 최신화했다.

## 4. 권장 운영 방식

- MPL 의존성은 가능하면 포크/직접 수정 없이 사용합니다.
- 부득이하게 수정할 경우, "수정 파일 목록 + 패치 + 소스 제공 위치"를 릴리즈 노트에 함께 남깁니다.
- 릴리즈마다 THIRD_PARTY_LICENSES.md와 본 문서를 같이 점검합니다.
